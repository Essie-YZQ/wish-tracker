# UI_Guidelines.md

## Purpose

This file defines the visual and interaction design system for Bloom Journal.
All AI agents must read this file before making any UI changes.

---

## Product Identity

**Project Name:** Bloom Journal

**Core Feelings:**
- Cozy
- Magical
- Garden Inspired
- Calm
- Delightful
- Rewarding

**Avoid:**
- Corporate Dashboard Style
- Bootstrap Look
- Generic Productivity App Appearance
- Flashy Gaming Effects

---

## Design References

Use these products as inspiration for emotional design, reward systems,
micro interactions, and visual tone. Do not copy them directly.

- Finch
- Forest
- Animal Crossing
- Stardew Valley
- Apple Journal

---

## Color Philosophy

**Preferred Colors:**
- Dusty Rose
- Sage Green
- Cream White
- Warm Beige

**Avoid:**
- Neon Colors
- Pure Black Backgrounds
- High Contrast Gaming Palettes

---

## Component Strategy

**Preferred Libraries:**
- shadcn/ui
- Framer Motion
- Magic UI

Reuse existing components whenever possible.
Avoid creating custom components when standard solutions already exist.

---

## Animation Philosophy

Animations should feel:
- Elegant
- Slow
- Intentional
- Rewarding

**Avoid:**
- Fast Spinning
- Aggressive Bounce Effects
- Casino Style Animations

---

## Bloom Journal Specific Rules

**Wish Pool:**
Should feel magical and valuable.

**Flower Unlock:**
Should create anticipation and surprise.

**Flower Collection:**
Should feel like building a personal garden.

**Achievements:**
Should feel earned rather than gamified.

---

## UI Decision Process

Before writing any code, always describe:

1. User Experience — what does the user feel?
2. Layout — how is it arranged?
3. Animation — how does it move?
4. Then write code.

Never jump directly into implementation.
